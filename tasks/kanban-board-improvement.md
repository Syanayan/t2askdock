# カンバンボード改善タスク

> 目的: スクリーンショットのような本格的なカンバンUIを実現する。
> 現状は最小限のDnDのみ実装済み。カード情報・カラム表現・ツールバーを強化する。

---

## 現状との差分サマリー

| 項目 | 現状 | 目標 |
|------|------|------|
| カード表示 | タイトルのみ | 番号・担当者・優先度・タグ・期限・説明 |
| カラムヘッダー | テキストのみ | 色帯 + 件数バッジ + メニュー |
| タスク追加 | 別コマンドのみ | カラム内「＋タスクを追加」ボタン |
| カード操作 | DnDのみ | 三点メニュー（編集・削除） |
| ツールバー | なし | フィルター・マイタスク・折り畳み |

---

## グループA: カード情報の強化

- [x] **A-1 タスク番号の表示**
  - 各タスクに連番（#1, #2, ...）を付けてカード左上に表示
  - `BoardTask` 型に `sequenceNumber?: number` を追加し、取得側で付番する
  - カード右上には「...」三点メニューボタンを配置する

- [x] **A-2 担当者アバターの表示**
  - `assignee` が設定されている場合、カード下部に丸型アイコンを表示
  - 名前の頭文字または人型SVGアイコンで表示（画像URL不要）
  - `assignee` が `null` の場合は非表示

- [x] **A-3 優先度バッジの表示**
  - `priority` をカード下部にバッジ表示
    - `low` → グレー「優先度:低」
    - `medium` → 青「優先度:中」
    - `high` → オレンジ「優先度:高」
    - `critical` → 赤「優先度:最高」
  - `medium` は表示しない（デフォルトのため省略可）

- [x] **A-4 タグバッジの表示**
  - `tags` 配列の各要素をカード下部にバッジ表示
  - バッジは丸角・色付き背景（タグ名のハッシュで色決定、または固定パレット）
  - タグが0件の場合は非表示

- [x] **A-5 期限日の表示**
  - `dueDate` が設定されている場合、カード下部に表示（例: `2025/07/18`）
  - 期限切れ（今日より前）は赤文字で表示

- [x] **A-6 説明プレビューの表示**
  - `description` が設定されている場合、タイトルの下に最大2行のプレビュー表示
  - CSS `line-clamp: 2` または `overflow: hidden` + 固定高で切り捨て

- [x] **A-7 サブタスクインジケーターの表示**
  - `parentTaskId` が `null` かつサブタスクを持つ場合、カード下部に `≡` アイコンを表示
  - サブタスク件数は現状取得できないため、`boardTask.hasChildren?: boolean` フラグで対応

- [x] **A-8 カードの三点メニュー**
  - カード右上の「...」ボタンクリックで操作メニューを表示
  - 選択肢: 「編集」→ `taskDock.updateTask` コマンド呼び出し / 「削除」→ `taskDock.deleteTask` コマンド呼び出し
  - `vscode.postMessage({ type: 'card:menu', action: 'edit'|'delete', taskId })` で拡張機能側へ委譲

---

## グループB: カラム表示の改善

- [x] **B-1 カラムヘッダーの色帯**
  - 各カラム上部に幅広の色帯（高さ4px程度のボーダー）を追加
    - `todo` → 青 (#2196F3)
    - `in_progress` → 緑 (#4CAF50)
    - `blocked` → 赤 (#F44336)
    - `done` → ピンク/パープル (#9C27B0)
  - `border-top` または 疑似要素 `::before` で実装

- [x] **B-2 カラム件数バッジ**
  - カラムタイトルの右横にタスク件数を表示（例: `Todo 3`）
  - タスクのDnD後にリアルタイム更新する（再`render()`時に件数を再計算）

- [x] **B-3 カラム内「＋タスクを追加」ボタン**
  - 各カラムのタスクリスト上部に「+ タスクを追加」ボタンを配置
  - クリックで `vscode.postMessage({ type: 'card:create', status })` を送信
  - 拡張機能側で `taskDock.createTask` を呼び出し、作成後にWebviewを更新する

- [ ] **B-4 カラムの三点メニュー**
  - カラムヘッダー右の「...」ボタンクリックで操作メニューを表示
  - 選択肢: 「このカラムをすべて完了にする」など（将来拡張用、現状はプレースホルダーでOK）

## グループC: ツールバー / フィルター

- [ ] **C-1 フィルターバーの追加**
  - ボードタイトルの下にツールバー行を追加
  - 「フィルター」「マイタスク」「アーカイブ」の各ボタンを配置（非機能ボタンはグレーアウト）

- [ ] **C-2 マイタスク絞り込み**
  - 「マイタスク」ボタンクリックで `assignee === 'system'` のタスクのみ表示（現状ユーザーは'system'固定）
  - JS 側の `tasks` フィルタリングのみで実装（バックエンド呼び出し不要）

- [ ] **C-3 折り畳みボタン**
  - 「折り畳む」ボタンクリックで全カラムの説明・バッジ類を非表示にし、タイトルのみ表示
  - CSS クラスの付け外しで実装（`body.collapsed` など）

## グループD: タスクインライン作成（任意）

## 技術上の注意点

- Webview内のDOMはサンドボックス化されているため、`<details>` / CSS クラス切り替えなどクライアントサイドJSで完結させる
- `vscode.postMessage` → `panel.webview.onDidReceiveMessage` のパターンは既存 `board:drop` で実績あり
- タグ色のハッシュ関数は `tag.charCodeAt(0) % palette.length` 程度のシンプルな実装で十分
- カード番号は `tasks.indexOf(task) + 1` ではなくDBの挿入順を保持する `sequenceNumber` が望ましいが、現状スキーマにないため一旦表示順インデックスで代替可
