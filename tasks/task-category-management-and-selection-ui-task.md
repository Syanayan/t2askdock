# カテゴリー管理 / 選択表示改善タスク

> 目的: カテゴリーの削除を行わず、名称変更と表示制御で運用する。現在選択中の項目を視覚的に分かりやすくする。

---

## 目的

- カテゴリー削除を実装しない
- カテゴリー追加・名称変更・非表示/Archive を可能にする
- ツリー表示の現在選択項目を明確化する
- 選択状態とステータス表示を混在させない

---

## 実装対象

- `src/ui/webview/category-webview-panel.ts`
- `src/ui/tree/tree-view-controller.ts`
- `src/ui/webview/task-table-webview-panel.ts`
- `src/core/usecase/create-category-usecase.ts`
- `src/core/usecase/update-category-usecase.ts`

---

## 要件

- カテゴリー削除ボタンは実装しない
- カテゴリー追加はカテゴリー一覧画面の右上ボタンで行う
- カテゴリー追加時は名称のみ入力する
- カテゴリー名称変更を可能にする
- 必要に応じてカテゴリーを非表示または Archive できる設計にする
- `Archive` 操作はカテゴリー一覧・タスク一覧の右上に配置
- 選択中のタスク/カテゴリーは緑色強調表示する
- 状態表示（ステータス）と選択表示のビジュアルを分離する

---

## グループA: カテゴリー追加・編集

- [x] **A-1 追加ボタン配置**
  - カテゴリー一覧画面の右上に `Add Category` ボタンを設置
  - ボタン押下で VS Code ネイティブ `showInputBox` ダイアログを開く
  - 空文字・キャンセル時は何もしない

- [x] **A-2 名称変更 UI**
  - 既存カテゴリー行に `Rename` ボタンを追加
  - ボタン押下で VS Code ネイティブ `showInputBox` ダイアログを開く
  - 空文字・キャンセル時は何もしない

- [x] **A-3 非表示 / Archive モード設計**
  - `Archive Category` ボタンをヘッダーに設置
  - 選択タスクの `projectId` を元にカテゴリーを Archive（`archived = 1`）に設定
  - Archive されたカテゴリーは DB から取得されないため一覧から除外

---

## グループB: 選択表示改善

- [x] **B-1 現在選択項目のハイライト**
  - タスク一覧で現在選択中の行を緑色で強調（`outline:2px solid #2e7d32`）
  - 選択とステータスが重ならないよう、枠線・背景・文字色を分離

- [x] **B-2 判定ロジックの修正**
  - 選択状態は `tr.selected` クラスで管理
  - ステータス表示は `.status-done` 等の CSS クラスで別管理

- [x] **B-3 連携整合性**
  - 選択変更時に `applySelection()` が即時 UI 更新
  - Shift/Ctrl マルチ選択に対応

---

## 受け入れ基準

1. カテゴリー一覧に `Add Category` ボタンがあり、名称のみで追加できる
2. カテゴリー名を変更できる UI が存在する
3. 選択中のタスク/カテゴリーは緑色で視認できる
4. 選択表示とステータス表示が混在しない
5. カテゴリー削除を実装していない
