# カテゴリー管理 / 選択表示改善タスク

> 目的: カテゴリーの削除を行わず、名称変更と表示制御で運用する。現在選択中の項目を視覚的に分かりやすくする。

---

## 目的

- カテゴリー削除を実装しない
- カテゴリー追加・名称変更・非表示/Archive を可能にする
- ツリー表示の現在選択項目を明確化する
- 選択状態とステータス表示を混在させない

---

## 実装対象

- `src/ui/webview/category-webview-panel.ts`
- `src/ui/tree/tree-view-controller.ts`
- `src/ui/webview/task-table-webview-panel.ts`
- `src/core/usecase/create-category-usecase.ts`
- `src/core/usecase/update-category-usecase.ts`

---

## 要件

- カテゴリー削除ボタンは実装しない
- カテゴリー追加はカテゴリー一覧画面の右上ボタンで行う
- カテゴリー追加時は名称のみ入力する
- カテゴリー名称変更を可能にする
- 必要に応じてカテゴリーを非表示または Archive できる設計にする
- `Archive` 操作はカテゴリー一覧・タスク一覧の右上に配置
- 選択中のタスク/カテゴリーは緑色強調表示する
- 状態表示（ステータス）と選択表示のビジュアルを分離する

---

## グループA: カテゴリー追加・編集

- [ ] **A-1 追加ボタン配置**
  - カテゴリー一覧画面の右上に `Add Category` ボタンを設置
  - ボタン押下で `name` だけを入力するダイアログまたは Webview を開く
  - `Ctrl + Enter` で追加実行できる

- [ ] **A-2 名称変更 UI**
  - 既存カテゴリー行に `Rename` もしくは `Edit` 操作を追加
  - 変更した名称を保存できるフォームを表示
  - 保存時は空文字を拒否する

- [ ] **A-3 非表示 / Archive モード設計**
  - カテゴリーの非表示は `isHidden` フラグなどで制御する
  - `Archive` はカテゴリー一覧で対象を選択して実行できるようにする
  - `Archive` 対象のカテゴリーは通常ツリー/一覧から除外される設計を検討

---

## グループB: 選択表示改善

- [ ] **B-1 現在選択項目のハイライト**
  - ツリー表示・タスク一覧で現在選択中の項目を緑色で強調
  - 選択とステータスが重ならないよう、枠線・背景・文字色を分離

- [ ] **B-2 判定ロジックの修正**
  - 現在の色判定が逆になっている場合は修正する
  - 表示中の「選択状態」と「ステータス状態（例: Done/Review）」を別々に評価する

- [ ] **B-3 連携整合性**
  - ツリー選択時の対象 ID が Webview / コマンド実行に正しく伝搬することを確認
  - 選択中の項目が変更されたときに UI が即時更新されること

---

## 受け入れ基準

1. カテゴリー一覧に `Add Category` ボタンがあり、名称のみで追加できる
2. カテゴリー名を変更できる UI が存在する
3. 選択中のタスク/カテゴリーは緑色で視認できる
4. 選択表示とステータス表示が混在しない
5. カテゴリー削除を実装していない
