# エディタサブタスク表示 & カンバンUI改善タスク

> 目的: エディタのタスクリストに階層（サブタスク）表示を導入し、
> カンバンの追加導線とメニューUIをダークテーマ対応のうえで操作性を向上させる。

---

## 現状との差分サマリー

| 項目 | 現状 | 目標 |
|------|------|------|
| エディタのリスト | フラット表示中心 | 親子階層を展開/折り畳みで表示 |
| 階層の見た目 | 判別しづらい | 子要素にインデントを付けて可視化 |
| カンバンのタスク追加 | クリック中心 | `Ctrl + Enter` でも追加可能 |
| 空入力時の追加 | 不明確 | 空なら追加せずキャンセル |
| カンバンの「...」UI | テーマ追従が不十分 | ダークテーマに統一 |
| カンバンの「...」機能 | 限定的 | ツリーメニュー項目を選択可能 |

---

## グループA: エディタのサブタスク表示

- [x] **A-1 ツリー構造データの受け渡し確認**
  - エディタリスト描画時に `children`（または `parentTaskId` ベースで再構成した子配列）を扱えるようにする
  - 子要素を持つ行かどうかを判定するフラグ（例: `hasChildren`）を明示する

- [x] **A-2 展開/折り畳みトグルの追加**
  - 子を持つ行の先頭にトグルボタンを表示
    - 折り畳み時: `▶`
    - 展開時: `▼`
  - ボタン押下時のみ直下の子要素を表示/非表示に切り替える
  - 行クリック全体ではなく、トグル押下を起点に状態変更する

- [x] **A-3 展開状態の管理**
  - フロント側 state（例: `expandedTaskIds: Set<string>`）で開閉状態を保持
  - 再描画時に状態を引き継ぐ（不要な全展開/全折り畳みを避ける）

- [x] **A-4 子要素のインデント表示**
  - 親より左余白（インデント）を追加して階層を明示
  - 深さに応じて段階的に余白を増やす（例: `depth * 12px`）

- [x] **A-5 直下のみ表示ルールの明確化**
  - 親を展開した時に直下の子のみ表示し、孫は子側のトグルで制御
  - 折り畳み時は配下ノードをまとめて非表示

---

## グループB: カンバン「タスク追加」挙動の改善

- [x] **B-1 ダークテーマ対応（追加UI）**
  - タスク追加入力欄・追加/キャンセル導線の背景色/文字色/ボーダーをテーマ変数に合わせる
  - ライト/ダーク双方で視認性を確認

- [x] **B-2 `Ctrl + Enter` でタスク追加**
  - タイトル入力中に `Ctrl + Enter` 押下で追加処理を実行
  - `keydown` ハンドラで対象キーを判定し、既存追加処理を呼び出す

- [x] **B-3 空入力時キャンセル**
  - タイトルが空文字または空白のみの場合は作成しない
  - 追加操作をキャンセル扱いにして入力UIを閉じる

- [x] **B-4 既存操作との整合**
  - Enter / Escape / ボタンクリックとの競合を整理
  - 重複送信防止（連打時の二重追加防止）を確認

---

## グループC: カンバン「...」ボタン改善

- [x] **C-1 ダークテーマ対応（三点メニュー）**
  - メニューボタン本体、ポップアップ、ホバー、区切り線をダークテーマに調整
  - VS Code のテーマカラー（CSS変数）優先で実装

- [x] **C-2 ツリー右クリックメニュー項目の連携**
  - ツリー構造で右クリック時に表示されるメニュー項目と同等の選択肢を「...」に表示
  - 既存コマンドIDを再利用し、カンバンから実行可能にする

- [x] **C-3 メニュー項目の表示条件統一**
  - タスク種別/状態による表示可否（例: 親タスクのみ表示、完了時は無効）をツリーと揃える
  - コマンド実行時の対象タスクID受け渡しを統一

- [x] **C-4 アクセシビリティ/操作性確認**
  - キーボード操作（上下移動・Enter確定・Esc閉じる）を確認
  - フォーカスアウト時にメニューを閉じる

---

## 着手順（最小動線優先）

`A-1 → A-2 → A-4 → A-3 → A-5 → B-2 → B-3 → B-1 → C-1 → C-2 → C-3 → C-4`

まずサブタスクの階層表示を成立させ、その後にカンバン追加操作と三点メニューの改善を行う。

---

## 技術上の注意点

- エディタとカンバンでコマンド定義が二重化しないよう、可能な限り既存コマンドIDを共通利用する
- テーマ対応は固定色を避け、`--vscode-*` 系CSS変数を使って実装する
- キーイベントは入力中のみ有効化し、グローバルショートカットとの衝突を避ける
- 開閉状態はセッション内保持を基本とし、永続化が必要かは別タスクで判断する
