# 要求仕様書 v1.1
## VS Code拡張機能：オフライン共有タスク管理ツール

- 作成日: 2026-04-24
- 版数: v1.1

## 1. 概要

### 1.1 目的
オフライン環境で、サーバーを使わずに共有フォルダ上のDBを利用し、VS Code上でプロジェクト単位のタスク管理を行える拡張機能を提供する。

### 1.2 前提
- インターネット接続不要で主要機能が動作すること
- 外部サーバーを必須としないこと
- 複数人が同一DBを参照・更新できること

## 2. スコープ

### 2.1 対象（In Scope）
1. VS Code拡張機能として実装
2. オフライン・サーバーレス運用
3. 共有フォルダ上DBの利用
4. プロジェクト別タスク表示
5. Tree表示 / Board表示（ドラッグ＆ドロップ対応）
6. タスクCRUD、検索、フィルタ、ソート
7. 管理者＋アクセスキーによる利用制御
8. 複数DB場所の選択・切替
9. ライト/ダークモード対応
10. Activity Barボタン・コマンドパレット対応
11. 将来拡張（GitHub/GitLab連携、ローカルAI連携）

### 2.2 非対象（Out of Scope）
- クラウド必須同期
- 常駐Webサーバー
- SaaS必須の認証基盤

## 3. 利用者と権限

### 3.1 利用者区分
- 管理者: 初回DB作成者。キー発行・失効・権限管理を実施
- 一般ユーザー: アクセスキー認証後、許可範囲で閲覧/編集

### 3.2 認証要件
- 共有DB利用時にアクセスキー入力を必須とする
- キーやトークンの平文保存は禁止
- SecretStorage等の安全な保存方式を利用する

## 4. 機能要求（FR）

### FR-01 タスク管理
- タスク作成・編集・削除・複製
- 主項目: タイトル、説明、ステータス、優先度、担当者、期限、タグ、親子タスク

### FR-02 表示機能
- Tree View: 階層表示（プロジェクト > タスク > サブタスク）
- Board View: カンバン表示（列=ステータス）
- Board上のドラッグ＆ドロップでステータス変更

### FR-03 プロジェクト単位表示
- プロジェクト切替で対象タスクを表示
- ワークスペース単位の表示切替に対応

### FR-04 検索・絞り込み・並び替え
- 検索（タイトル/説明/タグ）
- フィルタ（ステータス、担当者、期限、優先度）
- ソート（更新日、期限、優先度）

### FR-05 VS Code統合
- Activity Barに起動ボタンを表示
- コマンドパレットから表示・操作可能
- 例: Open Task Board / Open Task Tree / Select Database / Create Task

### FR-06 DB選択・切替
- DB配置フォルダを任意選択可能
- 複数DB登録・切替をサポート

### FR-07 共有・同時編集
- 共有フォルダDBの同時利用を許容
- 競合対策（ロックまたは楽観ロック）を実装

### FR-08 監査・履歴
- 更新者、更新日時、変更概要を記録

## 5. 非機能要求（NFR）

### NFR-01 オフライン完全性
- ネット未接続でも全コア機能が利用可能

### NFR-02 軽量性
- 主要画面の初期表示は3秒以内（標準開発PC想定）
- 体感遅延の少ない操作性を維持

### NFR-03 信頼性
- 異常終了時のデータ破損を最小化
- 自動バックアップ（世代管理）を提供

### NFR-04 UI/UX
- モダンで使いやすいデザイン
- ライト/ダーク両モードで視認性を確保

### NFR-05 保守性・拡張性
- コア機能と連携機能を疎結合化
- Provider/Adapter方式を採用

## 6. 拡張要件A：GitHub/GitLab Issue連携

### 6.1 基本方針
- 初期リリースでは機能フラグで無効化可能
- コア機能に影響を与えない独立モジュールとして実装

### 6.2 要件
1. GitHub/GitLabの両Providerを追加可能
2. 手動同期を基本とする
3. オフライン時は同期停止しローカル作業継続
4. 競合時はローカル優先/外部優先/マージを選択可能
5. 同期ログを記録する

## 7. 拡張要件B：ローカルAI連携

### 7.1 基本方針
- ローカルAI（オンプレLLM）接続を前提とする
- クラウドAIを必須にしない

### 7.2 AIユースケース
1. 文章からタスク候補抽出
2. 優先度・期限提案
3. 重複タスク検出
4. 作業ログから進捗提案

### 7.3 制約
- AI提案は自動確定せず、ユーザー承認後に反映
- AI未接続時も通常機能に影響を与えない
- 入力データの外部送信を行わない

## 8. 受け入れ基準（Acceptance Criteria）

- オフライン環境で全コア機能が動作する
- サーバー不要で複数人が共有DBを利用できる
- 管理者作成・アクセスキー運用が成立する
- VS Codeのボタンとコマンド両方から画面表示できる
- プロジェクト単位の表示ができる
- Tree/Board両表示に対応する
- Boardのドラッグ＆ドロップで状態更新できる
- ライト/ダークモードに対応する
- 連携機能を無効化した状態でもコア機能が完全動作する

## 9. 推奨リリース計画

- Phase 1: コア機能（オフライン、認証、Tree/Board）
- Phase 2: 競合解決と監査ログ強化
- Phase 3: GitHub/GitLab連携
- Phase 4: ローカルAI提案機能
